# 下一批推广平台

目标不是“到处发广告”，而是把 ApiZero 放到开发者已经信任的地方。

## 平台优先级

| 优先级 | 平台 | 目的 | 发布内容 | 登录/人工 |
| --- | --- | --- | --- | --- |
| P0 | Apifox API Hub | 国内 API 搜索和调试信任 | 导入 OpenAPI，发布公开 API 文档 | 需要登录 |
| P0 | Postman Public API Network | 国际/工具流量 | Public Workspace + Collection | 需要登录 |
| P1 | 掘金 | 技术文章分发 | 天气、条码、QQ、热搜教程 | 需要登录 |
| P1 | 知乎 | 问答长尾 | 回答“免费天气 API/条码 API/QQ 头像 API” | 需要登录 |
| P1 | V2EX | 冷启动反馈 | 求测帖，不硬广 | 需要登录 |
| P2 | SegmentFault / 开源中国 | 长尾收录 | 改写 CSDN 稿件 | 需要登录 |
| P2 | Gitee / GitCode | 国内代码搜索 | 镜像 GitHub 仓库 | 需要登录 |

## 一句话策略

- Apifox/Postman：把 `openapi.json` 变成可信公开文档。
- 掘金/CSDN：持续发可运行教程。
- 知乎：回答已有问题，不开新广告问题。
- V2EX：求反馈、求测试，不讲“我们很强”。
- Gitee/GitCode：镜像仓库，吃国内代码搜索流量。

## 本周执行顺序

1. 发布 Apifox API Hub。
2. 发布 Postman Public Workspace。
3. 掘金发 3 篇：天气、条码、QQ。
4. 知乎回答 5 个问题：免费天气 API、商品条码查询 API、QQ 头像 API、API 聚合平台、Node.js 调天气。
5. V2EX 发一次求测帖。
6. Gitee 镜像 `apizero-examples`。

## 复盘指标

- GitHub：访问、Star、Issue。
- Apifox/Postman：浏览、Fork/Run。
- 文章平台：阅读、收藏、评论。
- 官网：referrer 里来自平台的新增访客。
- API：对应接口调用量是否上涨。
